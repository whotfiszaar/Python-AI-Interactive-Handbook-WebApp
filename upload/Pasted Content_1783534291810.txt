client = OpenAI(

PYODIDE INTEGRATION (usePyodide hook)
- Load Pyodide from CDN on first visit to playground
- Show loading progress ("Loading Python runtime... 45%")
- Cache the Pyodide instance in module-level variable (never reload)
- Execute code by calling pyodide.runPythonAsync(code)
- Capture stdout by redirecting pyodide's stdout to a JS callback
- Capture stderr for error messages
- Set a 10-second execution timeout (terminate if exceeded)
- For "Python + AI" mode: detect if code contains "from openai import OpenAI" or "base_url", and if so, send the code to the /api/playground/run route instead of Pyodide

API ROUTE: POST /api/playground/run
- Receives: { code: string }
- Reads API key from Settings table
- If code contains OpenAI client pattern, extract the model and messages from the code string using regex
- Make the actual HTTP request to OpenRouter from the server side
- Return the response text
- If code does not contain OpenAI pattern, return error asking user to switch to Python Only mode
- Never expose the API key in the response

API ROUTE: POST /api/progress
- Receives: { dayNumber: number, completed?: boolean, bookmarked?: boolean, notes?: string }
- Upserts DayProgress record
- Returns updated record

API ROUTE: GET /api/progress
- Returns all DayProgress records

API ROUTE: POST /api/assessments
- Receives: { assessmentId: string, score: number, total: number, answers: object }
- Creates AssessmentScore record
- Returns created record

API ROUTE: GET /api/assessments
- Returns all AssessmentScore records

API ROUTE: POST /api/notebooks
- Receives: { name: string, cells: object }
- Creates Notebook record
- Returns created record

API ROUTE: GET /api/notebooks
- Returns all Notebook records

API ROUTE: PUT /api/notebooks/[id]
- Receives: { name?: string, cells?: object }
- Updates Notebook record
- Returns updated record

API ROUTE: DELETE /api/notebooks/[id]
- Deletes Notebook record

API ROUTE: GET/PUT /api/settings
- GET returns Settings record
- PUT receives: { darkMode?: boolean, fontSize?: number, apiKeys?: object }
- Updates Settings record

CONTENT: DAYS 16-25 AI AND LLM THEORY

Day 16: What is Artificial Intelligence? Examples from daily life: Siri predicting what you ask, YouTube recommending videos, Google Maps finding the fastest route, face unlock on phones. What AI can do (recognize images, translate languages, play games) and cannot do (feel emotions, have original thoughts, understand the world like humans).
Day 17: Machine Learning vs Deep Learning. ML: learning from examples without being explicitly programmed. Example: how YouTube learns that Aarav likes cricket videos because he watches them often. Deep Learning: ML using neural networks with many layers. Example: how face recognition works by breaking an image into tiny features. Difference: deep learning needs more data and compute but handles complex tasks better.
Day 18: Neural Networks. What a neuron is in a network (takes input, does math, gives output). How neurons connect in layers (input layer, hidden layers, output layer). Analogy: a group of friends each voting on whether a picture shows a cat or dog, each friend looking at different features (ears, tail, fur). No math. Just the concept.
Day 19: Transformers and Attention. Why older methods (RNNs) struggled with long sentences (they forgot the beginning). What attention means: when reading "The cat sat on the mat because it was tired", the word "it" pays attention to "cat" not "mat". Why this simple idea revolutionized AI. Transformers process all words at once instead of one by one.
Day 20: Large Language Models (LLMs). What they are: Transformers trained on massive amounts of text from the internet. How they work at a high level: they predict the next word given previous words. How this simple task leads to seemingly intelligent behavior. GPT means "Generative Pre-trained Transformer". Other LLMs: Gemini (Google), Claude (Anthropic), Llama (Meta), Mistral (Mistral AI), DeepSeek (DeepSeek).
Day 21: Tokens, Context Window, Embeddings. Tokens: how text is split before the model sees it. "Aarav" might be 1 token. "Unbelievable" might be 3 tokens (un, believ, able). Show examples of tokenization. Context window: the maximum number of tokens a model can process in one conversation. GPT-4 has 128K, some have 1M. Why it matters: longer conversations get cut off. Embeddings: words converted to lists of numbers. Similar words (car and vehicle) have similar numbers. This lets computers compare meaning mathematically.
Day 22: Prompt Engineering. What makes a good prompt vs bad prompt. Bad: "tell me about cars". Good: "Explain the difference between petrol and electric cars in 3 bullet points, for a 13-year-old". System prompts: instructions that set the AI's behavior. Few-shot: giving examples in the prompt. Zero-shot: no examples. Temperature: 0 = always picks most likely word (precise), 1 = more random (creative). Show same prompt at temperature 0 vs 1 with different outputs.
Day 23: Hallucinations, RAG, Fine-tuning. Hallucinations: when the AI confidently says something wrong (example: making up a book title that does not exist). Why it happens (LLMs predict plausible text, they do not look up facts). RAG (Retrieval-Augmented Generation): before answering, search a database of real documents, then give those documents to the LLM as context. This reduces hallucinations. Fine-tuning: take a pre-trained model and train it further on your specific data (example: train on medical texts to make a better medical AI). Differences: prompting is cheapest and fastest, RAG adds real data, fine-tuning changes the model itself.
Day 24: AI Agents and Tool Calling. What an agent is: an LLM that can decide to take actions, not just generate text. Analogy: a regular LLM is like a person who can only talk. An agent is like a person who can also use a phone, calculator, and computer. Tool calling: the LLM outputs a special format saying "I want to call the weather tool with city=Delhi" instead of guessing the weather. The system runs the tool and gives the result back. The LLM then gives a proper answer. This is how AI assistants actually work.
Day 25: Software Concepts for AI. REST API: a way for programs to talk to each other over the internet. Analogy: a restaurant menu. You (client) look at the menu (API documentation), tell the waiter (HTTP request) what you want, and get food back (response). JSON: a format for structuring data that looks like Python dictionaries. HTTP methods: GET (read data), POST (send data). Client vs Server: your Python code is the client, OpenRouter is the server. API key: like a membership card that proves you have permission. Environment variables: storing secrets like API keys in a separate file, not in your code.

Day 36: What is MCP (Model Context Protocol)? Why it exists (before MCP, every AI tool had its own way of connecting to LLMs). Compare to USB standard: before USB, every printer had a different cable. MCP is like USB for AI tools.
